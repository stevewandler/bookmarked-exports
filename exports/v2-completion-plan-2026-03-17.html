02 Module Completion Status

Status of each V2.0 module based on Jira ticket analysis and known issues. Modules marked "Needs Validation" require Steve or Patrick to confirm actual working state on prodv2.

All Roles
1. Dashboard
Needs Definition
Role-based home screen with key metrics. Librarian: lists awaiting review, books in orders, system alerts. Teacher: classroom-focused. Superintendent: district-wide overview (total lists, lists awaiting review, approved orders, dynamic alerts). Super Admin: full system-wide metrics.
Status: Working with real data. What metrics/widgets belong on each role's dashboard needs defining — this is a content decision, not a build gap. The dashboard renders and connects to data; the question is whether it shows the RIGHT metrics per Sarah's spec above.
Librarian
2. Library Management
Buggy
Add books by ISBN scan or CSV bulk upload. Manage book statuses (Pending Review / Allowed / Limited Access / Don't Show / Deleted).

Known Issues (Jira)

  • BOOK-837: CSV Upload — books import without metadata (flags, cover images, author info, ratings). Silent failure — librarian could upload 500 books and not realize none have cover images.
  • BOOK-879: Librarian bulk-upload returns 403 despite valid campus assignment.
  • Known issue (Sarah): Manual ISBN input field may be disabled — camera scanner is the current workaround.
To complete V2.0: Fix BOOK-837 (metadata on CSV upload) and BOOK-879 (403 on bulk upload). Validate whether manual ISBN input works or must be fixed.
Librarian + Superintendent
3. Compliance Review / Book Intelligence
Partial
Review flagged books using Book Intelligence data: risk tiers, AI summaries, content analysis (Violence/Language/Sexual/Scary), educational value assessments, theme tags, ban/challenge history. Assign vetting decisions. Manage restricted content. Superintendent has district-wide compliance oversight with full Book Intelligence cards.

Known Issues (Jira)

  • BOOK-894: Ban/challenge data currently hardcoded — needs real data from API.
  • BOOK-896: FlagsTable using hardcoded defaults instead of real ban/challenge data props.
  • BOOK-893: Backend needs to expose individual ban and challenge records with counts via book-service API.
  • BOOK-900: ContentAnalysis ban status card has hardcoded colors instead of dynamic status-based colors.
  • BOOK-650/649: Age Appropriateness data population and UI — status unclear.
To complete V2.0: Replace all hardcoded Book Intelligence data with real API data (BOOK-893, 894, 896, 900). Validate age appropriateness status. The compliance review workflow depends on real data — hardcoded demos won't survive migration.
Librarian + Teacher
4. Book Circulation
Buggy
Check books in/out to students. Manage overdue returns. Librarian has full circulation; Teacher has classroom-only checkout.

Known Issues (Jira)

  • BOOK-838: Circulation page book search returns no results and check-in fails for valid ISBNs. UAT — currently in testing.
To complete V2.0: BOOK-838 must pass UAT. Circulation is a core workflow — if librarians can't check books in/out, V2.0 is not shippable.
Librarian
5. Student & Guardian Data
Buggy
Configure student rosters and guardian access via ClassLink or SFTP integration.

Known Issues (Jira)

  • BOOK-878: SFTP onboarding dialog never displays password — blocks ALL SFTP connections.
  • BOOK-767: API errors on Students page for SFTP and ClassLink status.
  • BOOK-761: Librarian cannot assign users to campus/classroom — 500 error.
  • BOOK-912: OneRoster 1.1 CSV import rejects valid users.csv due to non-standard header validation.
To complete V2.0: Fix all 4 bugs. SFTP is the primary data integration path for districts — if it's broken, onboarding is blocked. This module has the highest bug density of any V2.0 module.
Librarian + Superintendent
6. Book Orders (Kanban)
Partial
Librarian: Create book lists, drag-and-drop between "Books to Order" / "Books to Review" / "Book Orders", submit for admin review. Superintendent: Review book lists, see Book Intelligence data, drag between Pending/Approved, approve/reject with feedback notes.

Known Issues (Jira)

  • BOOK-904: CreateListDialog — books found via search are not saved to the created list.
  • BOOK-688: Book list creation incorrectly applies quantity multiplier.
  • BOOK-884: Teacher can only see/manage their own lists in Order. UAT
To complete V2.0: Fix BOOK-904 (can't save books to lists — fundamental break) and BOOK-688 (quantity bug). BOOK-884 in UAT needs to pass. Multiple enhancement stories (BOOK-897, 898, 907, 908) may also be needed for basic functionality.
Librarian + Superintendent
7. Reports
Needs Validation
Generate and export analytics reports: Book Usage, Review Activity, Compliance Summary. Librarian: school-level scope. Superintendent: district-wide, schedule automated delivery, add up to 10 recipients. Export: PDF/CSV/Excel.
To validate: Are reports actually built? The report types Sarah lists (Book Usage, Review Activity, Compliance Summary) — do they exist and pull real data? Can they be exported? Scheduled delivery is likely V2.1 if not built, but basic report generation and export should be V2.0.
Librarian
8. Book Search
Needs Validation
Search internal catalog by title/author/ISBN. Fallback to Open Library API. Add books from search results.
To validate: Does internal catalog search work? Does Open Library API fallback work? Can books be added from search results? If the manual ISBN field is disabled (per Sarah's known issue), search becomes the primary way to add individual books.
Librarian

Known Issues (Jira)

To complete V2.0: BOOK-910 must be completed — notes need to persist via the API. Public/private toggle (BOOK-901/902) can be V2.1.

03 Sprint Process for V2.0 Completion

How Sarah should structure sprints to systematically complete remaining V2.0 work. No new features until this list is green.

1 Triage & Validate (Sprint 0)

Before sprinting, validate every module marked "Needs Validation" above. Steve or Patrick walks through prodv2 and confirms:

  • Reports — are they built? What report types exist?
  • Book Search — does internal catalog search work? Open Library fallback?
  • Policy Management — can superintendents upload policies?
  • User Settings — does MFA work? Google SSO?
  • Parent Portal — does parent login + student activity view work?
  • Dashboard — are the right metrics showing per role?

This triage converts every "Needs Validation" to either "Complete" or "V2.0 Sprint Work."

2 Bug Fix Sprint (Critical Path)

Fix the 11 open bugs. Priority order based on migration impact:

  • Tier 1 — Blocks onboarding: BOOK-878 (SFTP password), BOOK-877 (user creation), BOOK-761 (user assignment), BOOK-912 (CSV import)
  • Tier 2 — Blocks core workflow: BOOK-838 (circulation search), BOOK-837 (CSV metadata), BOOK-879 (bulk upload 403), BOOK-829 (classroom creation)
  • Tier 3 — Compliance/UX: BOOK-776 (lock icons), BOOK-835 (library badges), BOOK-688 (quantity multiplier), BOOK-904 (list save), BOOK-767 (API errors)

3 Data Integrity Sprint

Replace all hardcoded/demo data with real API data:

  • BOOK-893: Expose ban/challenge records via API
  • BOOK-894: Replace hardcoded ban/challenge counts with real data
  • BOOK-896: FlagsTable real data props
  • BOOK-900: Dynamic ban status colors
  • BOOK-909: Connect book CRUD to book-service API
  • BOOK-910: Connect notes CRUD to notes-service API

4 UAT Pass Sprint

Get all UAT items through testing:

  • BOOK-838: Circulation search (currently in UAT)
  • BOOK-881/882/883: Book Intelligence barcode + copies (currently in UAT)
  • BOOK-884: Teacher order list scoping (currently in UAT)
  • BOOK-740: Login tracking to HubSpot (currently in progress)

5 V2.0 Complete Gate

Definition of "V2.0 Complete":

  • All 11 bugs resolved and verified
  • All hardcoded data replaced with real API data
  • All UAT items passed
  • All "Needs Validation" modules confirmed as working or fixed
  • Every module in this document shows green status
  • Patrick confirms migration readiness (V1 data transfer risks addressed)

Then: Begin V1→V2 migration for 140+ districts. Everything else is V2.1.
