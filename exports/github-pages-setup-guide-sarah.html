Publishing Branded HTML Exports with GitHub Pages

How to create shareable, professionally branded reports, dashboards, and documents using Claude Code + GitHub Pages. Takes ~10 minutes to set up, then every future export is automatic.

What You're Setting Up

A personal GitHub Pages site that lets you publish any HTML file as a shareable link — branded with Bookmarked's design system, no extra work required.

Component What It Does
GitHub Pages Repo A repository that automatically publishes HTML files as web pages
Brand CSS Bookmarked's master stylesheet — colors, fonts, cards, layouts. Your Claude uses this so everything looks on-brand
Claude CLAUDE.md Instructions A snippet you add to your project so Claude automatically publishes and brands exports

The result: You tell Claude "create a report on X" → Claude generates branded HTML → pushes to GitHub Pages → gives you a shareable link. One prompt, professional output.

Setup Steps

Do these once. After that, every export is automatic.

1

Create Your GitHub Pages Repository

Ask Claude to do this for you. Copy and paste this prompt:

# Paste this into Claude Code:

Create a GitHub Pages repository for me. Here's what to do:

1. Create a new public repo called "bookmarked-exports" under my GitHub account
   (my GitHub username is sarah-elmassry)
2. Add an index.html that just says "Bookmarked Exports"
3. Enable GitHub Pages on the main branch
4. Clone it to /tmp/bookmarked-exports so you can push files to it

The repo URL will be:
https://sarah-elmassry.github.io/bookmarked-exports/

Note: If you already have a bookmarked-exports repo, skip this step. Just make sure it's cloned to /tmp/bookmarked-exports.

2

Add the Brand CSS to Your Repo

The brand CSS is the Bookmarked design system as a stylesheet. Ask Claude:

# Paste this into Claude Code:

Copy the Bookmarked brand CSS into my exports repo:

1. Download the CSS from this URL and save it:
   https://stevewandler.github.io/bookmarked-exports/design/bookmarked-brand.css
   Save to: /tmp/bookmarked-exports/design/bookmarked-brand.css

2. Also grab the Bookmarked logos:
   - Green logo (for light backgrounds):
     https://stevewandler.github.io/bookmarked-exports/design/bookmarked-logo-green.png
   - White logo (for dark/teal backgrounds):
     https://stevewandler.github.io/bookmarked-exports/design/bookmarked-logo-white.png
   Save to: /tmp/bookmarked-exports/design/

3. Commit and push everything.
3

Add Export Instructions to Your CLAUDE.md

This tells Claude how to handle all HTML output going forward. Add this block to your project's CLAUDE.md file:

Add to CLAUDE.md

## HTML Export Convention

When creating HTML reports, dashboards, or documents:

1. **Brand CSS** — Always link the Bookmarked brand stylesheet:
   `<link rel="stylesheet" href="../design/bookmarked-brand.css">`
   Or embed the CSS inline for standalone files.
   Reference: https://sarah-elmassry.github.io/bookmarked-exports/design/bookmarked-brand.css

2. **Use brand classes** — Use the `bm-` prefixed CSS classes:
   - `.bm-hero` for page headers (teal gradient)
   - `.bm-card` for content cards
   - `.bm-badge--success`, `.bm-badge--warning` for status indicators
   - `.bm-stats-row` + `.bm-stat-card` for metrics
   - `.bm-table` for data tables
   - `.bm-cta-button` for action buttons
   - `.bm-eyebrow` for section labels

3. **Logo** — Always use the actual logo image, never text:
   `<img src="../design/bookmarked-logo-green.png" height="32">`
   Use white version on dark/teal backgrounds.

4. **Publish to GitHub Pages:**
   - Save file to `/tmp/bookmarked-exports/[category]/[filename].html`
   - Categories: reports, exports, design, proposals
   - `cd /tmp/bookmarked-exports && git add . && git commit -m "description" && git push`
   - Present URL: `https://sarah-elmassry.github.io/bookmarked-exports/[category]/[filename].html`

5. **Also open locally** — After publishing, run `open [filepath]` so I can see it immediately.

6. **Font** — Always use Urbanist (Google Fonts import):
   `@import url('https://fonts.googleapis.com/css2?family=Urbanist:wght@400;500;600;700;800&display=swap');`
4

Test It

Ask Claude to create a test page to verify everything works:

# Paste this into Claude Code:

Create a test HTML page to verify my GitHub Pages setup works.
Make it a simple branded page with:
- A hero section with the Bookmarked logo
- A card with some sample text
- A stats row with 3 metrics
Publish it to my bookmarked-exports repo and give me the link.

If you see a branded page at your GitHub Pages URL, you're done.

After setup, you just ask Claude to create things. It handles the branding and publishing.

Setup Verification Checklist

  • GitHub repo bookmarked-exports created and public
  • GitHub Pages enabled (Settings → Pages → Source: main branch)
  • Repo cloned to /tmp/bookmarked-exports
  • bookmarked-brand.css in the design/ folder
  • Both logo PNGs in the design/ folder
  • CLAUDE.md updated with export instructions
  • Test page loads at your GitHub Pages URL

Troubleshooting

404 after pushing

GitHub Pages takes 1-2 minutes to deploy after a push. Wait a minute and refresh. You can check deploy status by asking Claude: gh api repos/sarah-elmassry/bookmarked-exports/pages

CSS not loading / unstyled page

Make sure the CSS link path is correct relative to your file. If your file is in /exports/, the CSS is at ../design/bookmarked-brand.css. Or use the full URL: https://sarah-elmassry.github.io/bookmarked-exports/design/bookmarked-brand.css

Can't push / authentication error

Ask Claude to check your GitHub auth: gh auth status. If not authenticated, run gh auth login and follow the prompts.

Repo not cloned

If Claude says /tmp/bookmarked-exports doesn't exist, just ask it: "Clone my bookmarked-exports repo to /tmp/bookmarked-exports"
