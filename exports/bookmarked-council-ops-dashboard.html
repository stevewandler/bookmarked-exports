Engagement Signals

  • ✓ Session attendance: Present / absent / async follow-up
  • ✓ Portal feedback: Items submitted through the Content Scanner portal

  • ✓ Survey completion: May structured surveys
  • ✓ Slack activity: Messages, reactions, questions
  • ✓ Ad hoc input: Emails, conversations, NDA review notes
Engagement Tiers

How We Classify Members

  • Active 3+ engagement signals per sprint. Continued membership guaranteed.
  • Quiet 1–2 signals. Natasha reaches out — offer async options.
  • Silent 0 signals for 2+ sprints. Steve decides: re-engage or release seat.

Goal: highlight who's really engaged and encourage participation. Active members get first access benefits. Silent members risk losing their seat.

Natasha's Workflow

How Natasha Operates This via Claude

Natasha uses Claude Code to manage Content Scanner program operations. Key commands:

  • → "Update partner status" → Claude updates Confluence partner roster
  • → "Log meeting notes" → Claude creates Confluence page from template
  • → "Process this feedback" → Claude categorizes, adds to tracker, routes
  • → "Send session reminder" → Claude drafts Slack message for approval
  • → "What needs attention?" → Claude scans for stale items, deadlines
  • → "Engagement report" → Claude pulls portal votes + feedback + Slack activity
  • → "Draft monthly update" → Claude generates impact report
